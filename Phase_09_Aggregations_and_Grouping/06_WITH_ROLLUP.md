# 📈 Topic 9.6: The WITH ROLLUP Modifier

Standard reports show totals for each group. But many business reports also require "Sub-totals" and "Grand Totals" at the bottom. Formatting these in application code is tedious. MySQL provides the **`WITH ROLLUP`** modifier to generate these super-aggregate rows automatically.

---

## 1. Definition

**`WITH ROLLUP`:** An extension of the `GROUP BY` clause that generates additional "summary" rows representing higher-level aggregations (sub-totals and a grand total).

---

## 2. Why This Concept Exists

If you group by `City`, you get a row for Mumbai, a row for Delhi, etc. But usually, your boss also wants a row at the bottom showing the `TOTAL` for all cities combined. `WITH ROLLUP` injects this "Total" row into your result set natively.

---

## 3. How It Works (Hierarchy of Aggregation - PRO LEVEL)

When you group by multiple columns, `ROLLUP` generates rows for every level of the hierarchy.

If you have: `GROUP BY Country, City WITH ROLLUP`
MySQL will generate:
1.  Totals for every **City** (The detailed level).
2.  Sub-totals for every **Country** (Across all its cities).
3.  A **Grand Total** for everything! (Across all countries).

These super-aggregate rows are identified by having `NULL` in the columns that were "rolled up."

---

## 4. Syntax / Implementation

```sql
-- Single column rollup (Grand Total)
SELECT Department, SUM(Salary)
FROM Employees
GROUP BY Department WITH ROLLUP;

-- Multi-column rollup (Sub-totals + Grand Total)
SELECT Region, City, SUM(Sales)
FROM Sales_Data
GROUP BY Region, City WITH ROLLUP;
```

---

## 5. Identifying the ROLLUP Rows (GROUPING Function)

Because the summary rows contain `NULL`, it's hard to distinguish a "Total Row" from a row that just happened to have NULL in the data. Senior developers use the `GROUPING()` function to identify them:

```sql
SELECT 
    IF(GROUPING(Department), 'Company Total', Department) AS Dept_Label,
    SUM(Salary)
FROM Employees
GROUP BY Department WITH ROLLUP;
```

---

## 6. Real-Life Examples

**The Sales Monthly Report:**
```sql
SELECT 
    YEAR(Order_Date) AS Year,
    MONTH(Order_Date) AS Month,
    SUM(Total_Amount) AS Revenue
FROM Orders
GROUP BY Year, Month WITH ROLLUP;
-- Result will show:
-- 2023 | 01 | $10,000
-- 2023 | 02 | $15,000
-- 2023 | NULL| $25,000  <-- Sub-total for 2023
-- NULL | NULL| $25,000  <-- Grand Total for whole table
```

---

## 7. Common Mistakes

*   **Using ORDER BY with ROLLUP:** You cannot use `ORDER BY` and `WITH ROLLUP` together in the same query because `ROLLUP` produces rows in a specific hierarchical order. If you need to sort the results, you must use a subquery.
*   **Relying on positional NULLs:** Don't just check `IF column IS NULL` to find the total row; use the `GROUPING()` function to be absolutely sure the NULL was generated by the ROLLUP.

---

## 8. Tips & Best Practices (Pro-Level)

**ROLLUP for Dashboarding:**
`WITH ROLLUP` is extremely efficient for backend APIs powering "Accordion" style dashboards where you can see a summary and then "drill down" into the details. One query provides both levels of data!
